OpenIdea

A Claude Code plugin for managing a product idea lifecycle end-to-end, file-based — from a raw vision to an OpenSpec technical draft.

Quick Start

/openidea:init → /openidea:capture → /openidea:evaluate → /openidea:plan → /openidea:spec-draft

The core loop — always applies, solo or team. Vision → captured idea → evaluated & ready → milestone → OpenSpec change.

/openidea:compile → /openidea:export → /openidea:spec-audit

Extended — situational, only when a client or an external repo-less team is in the loop. compile/export slot in between plan and spec-draft; spec-audit runs any time after.

Commands

CommandWhat it does
/openidea:initStructure a product vision → openidea/BRIEF.md
/openidea:captureCapture a new idea — free-form story, or extracted from a document
/openidea:evaluateReview captured ideas for overlap/conflict, recommend a status
/openidea:planArrange ready ideas into openidea/ROADMAP.md milestones
/openidea:spec-draftGenerate an OpenSpec proposal/design/tasks from a milestone
/openidea:compileCompile ready ideas into a client-facing proposal narrative
/openidea:exportPackage a milestone's technical content for external handoff
/openidea:spec-auditRead-only drift audit — OpenSpec vs. proposal/ideas

Principles

File-based

All state lives in markdown under openidea/ — diffable, readable without extra tools.

Human decides, Claude drafts

Claude structures and recommends; status changes always go through your review.

Nothing disappears

Status changes, files don't. Every decision is archived with a reason, never deleted.

Independent from OpenSpec

Only spec-draft writes into openspec/, and only when that tooling is already set up.
